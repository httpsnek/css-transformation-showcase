# AI SaaS Pricing Card Showcase

A futuristic glassmorphism pricing card built with **Vite** (Vanilla JS) and **SCSS**.

---

## Setup

```bash
# 1. Install dependencies
npm install

# 2. Start the development server (http://localhost:5173)
npm run dev

# 3. Build for production
npm run build

# 4. Preview the production build locally
npm run preview
```

## Project Structure

```
src/
├── main.js               Entry point — imports SCSS
└── scss/
    ├── main.scss         SCSS entry point (@use of all partials)
    ├── _variables.scss   Design tokens ($colors, $transitions, glass values)
    ├── _mixins.scss      Reusable mixins (glass, neon-text, neon-glow-button)
    ├── _base.scss        Global styles (:root, body, a, h1)
    ├── _layout.scss      Page layout (#app centering)
    └── _components.scss  Card, badge, price, features, CTA button
```

## How SCSS Is Transformed Into CSS

Vite uses the **Dart Sass** compiler (the `sass` devDependency) to process `.scss` files before bundling.
The transformation happens in three stages:

1. **Variables** — `$color-neon-cyan`, `$glass-blur`, etc. declared in `_variables.scss` are resolved at compile time. No variable names appear in the output CSS; every reference is replaced with its literal value.

2. **Mixins** — `@mixin glass { … }` and `@mixin neon-text($color) { … }` defined in `_mixins.scss` are expanded inline wherever `@include glass` or `@include neon-text(…)` appears. The mixin block disappears and its declarations are copied directly into the calling rule.

3. **Nesting** — Nested selectors such as `.card { &:hover { … } }` and `button { &:focus { … } }` are flattened by the compiler into standard CSS rules (`.card:hover { … }`, `button:focus { … }`).

The five partials are stitched together by `src/scss/main.scss` using `@use`, then Vite bundles and minifies the result into a single CSS file in `dist/assets/`.

## Source Maps

After running `npm run build`, the `dist/assets/` directory contains:

| File | Description |
|---|---|
| `index-<hash>.css` | Minified production stylesheet. The final line contains a `sourceMappingURL` comment pointing to the `.map` file. |
| `index-<hash>.css.map` | CSS source map (JSON, Source Map v3). Lists the original SCSS partials (`_variables.scss`, `_mixins.scss`, `_base.scss`, `_layout.scss`, `_components.scss`) as `sources`. |
| `index-<hash>.js` | Minified JavaScript bundle. |
| `index-<hash>.js.map` | JS source map generated by Vite's `build.sourcemap: true` option. |

> **Note:** Vite 7 does not generate external CSS source maps natively. The `.css.map` file is produced by the custom `cssSourceMapPlugin` defined in `vite.config.js`, which hooks into Rollup's `generateBundle` phase to emit the companion map file and append the `sourceMappingURL` reference to the CSS asset.
